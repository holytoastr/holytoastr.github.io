iHave To Go: Group project that uses Ruby on Rails, AngularJS, and Google Maps API to build a full-stack website to rate public restrooms based on accessibility and cleanliness.

The goal of this project was to have an AngularJS front-end pulling data from a Rails supplied JSON API on the back-end. To tackle this challenge, the team initially divided into specialties: front-end design, front-end Angular, back-end Rails, and API research. Each day, we reconvened to establish roles for the next sprint. In this way we were able to tackle a large number of issues in the limited time frame we were given to work with. When we encountered a challenge that one teammate was struggling with, we would pair to share brains and overcome the issue.

My main contribution to this project was the Rails API and the Angular controllers and routes. The greatest challenge for me was determining how to connect Angular and Rails in such a way that Rails could gracefully hand off control and allow the Angular configurations to control the front-end views.

Emotional Corgum: Uses Ruby on Rails, Postgres, and Devise to create a full-stack web-site that allows users to create persistent new content and comment on existing content.

For this rails project, I approached it by working on a single model at a time. Once that model was completely built out and all errors had been eliminated, I added the next model. In this way, I could follow an error driven development path. Each error guided me to the next step in developing the app. In the future, I would like to implement a way to allow users to view more than one "emotion" at a time.

Memory Game: Uses HTML, CSS, and pure JavaScript to create an interactive web-hosted memory game.

To approach this challenge, I broke the game down into its most basic component -- a card. The card had to have certain properties: It had to have a location on the board, it had to have a face, it had to have a back, and the user had to be able to flip the card over.

Once I had a card, I could move on to the next component: a deck of cards. These again had their own properties. The face cards had to be created in sets of pairs. The deck had to be shuffled. Etc...

Only after the entire deck was in place did I begin to add the game rules. There were some challenges I had to overcome in implementing these rules. Some of these were identifying which cards were clicked, remembering the values of the active clicked cards, and preventing the user from clicking matched cards

Tic Tac Toe: Uses HTML, CSS, and JQuery to create an interactive Tic Tac Toe board

The purpose of this project was to take advantage of jQuery to create and manipulate the DOM. In the future, I would like to add some logic to allow the game to determine and announce a winner.
